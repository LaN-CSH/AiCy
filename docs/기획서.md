# AiCy — AI 버튜버 프로젝트 기획서

> **AiCy = AI Cyber Project.** LLM을 두뇌로, VRM 아바타를 몸으로 삼아
> 실시간 라이브 채팅에 반응하며 방송하는 AI 버튜버.
>
> 문서 버전: v0.1 · 최종 수정: 2026-06-08 · 상태: 기획 단계 (코드 작성 전)

---

## 1. 한 줄 정의

**시청자의 라이브 채팅을 읽고 → AiCy가 자기 성격대로 답을 만들고 → 목소리로 말하며 → 아바타가 입을 맞춰 움직이는 것**을, 사람 개입 없이 자동으로 돌리는 방송 시스템.

---

## 2. 목표와 범위

### 하고 싶은 것 (In Scope)
- LLM 기반 자율 대화 (페르소나 일관성 유지)
- 라이브 채팅 실시간 수집 및 반응
- TTS 음성 출력 + 아바타 립싱크/표정
- OBS를 통한 실제 플랫폼 송출
- 채팅이 없을 때의 idle(혼잣말) 행동
- 욕설·위험 발언 안전 필터

### 지금은 안 할 것 (Out of Scope, 나중에)
- 게임 플레이 연동 (화면 보고 반응)
- 후원/도네이션 알림 반응 (4단계 이후)
- 노래/커버 (기존 yt-dlp 자산은 보관)
- 자체 음성 모델 학습 (당분간 ElevenLabs 등 외부 TTS 사용)

---

## 3. 플랫폼 전략 (결정됨)

**이중 언어 전략으로 간다.**

| 버전 | 플랫폼 | 이유 |
|---|---|---|
| **영어판 (1순위 착수)** | **YouTube Live** | 공식 API가 가장 잘 정리돼 있어 채팅 연동이 안정적. 글로벌 접근성 |
| **한국어판 (이후 확장)** | **치지직 (Chzzk)** | 한국어 시청자 타깃 1순위 플랫폼 |

> **설계 원칙:** 처음부터 **언어와 플랫폼을 설정값으로 분리**한다.
> 채팅 입력부와 페르소나 프롬프트를 "교체 가능한 부품"으로 만들어,
> YouTube/영어 → 치지직/한국어 전환이 코드 수정 없이 설정만으로 되게 한다.
>
> - 채팅 연동: `ChatSource` 인터페이스 하나에 YouTube 구현체 / Chzzk 구현체를 끼운다.
> - 송출: 두 플랫폼 모두 **OBS로 송출**하므로 송출 파이프라인은 공통.

## 4. 아바타 기술 (결정됨: 3D VRM)

조건은 **"코드로 다루기 쉽고 + 상업적 이용 가능한 무료 모델"**. → **3D VRM 채택.**

| 기준 | Live2D | **3D VRM (채택)** |
|---|---|---|
| 코드 제어 | Cubism SDK 필요, 상대적으로 폐쇄적 | `@pixiv/three-vrm` 오픈소스로 표정·립싱크·뼈대 전부 코드 제어 |
| 무료 상업 모델 | 무료 상업용 모델이 드묾 | **VRoid Studio로 직접 제작 → 내가 소유**, 라이선스 걱정 없음 |
| 라이선스 비용 | 매출 기준 유료 라이선스 존재 | 오픈 표준, SDK 비용 없음 |
| 송출 연동 | 별도 처리 | 브라우저에서 렌더 → **OBS 브라우저 소스로 그대로 캡처** |

**결정:** VRoid Studio(무료)로 AiCy 모델을 직접 제작하고, `three-vrm`(브라우저)으로 렌더링·제어한다.

## 5. 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                      Orchestrator (백엔드, Python)            │
│                                                              │
│  ChatSource ──▶ Brain(LLM+페르소나+기억) ──▶ 답변 텍스트      │
│  (YouTube/Chzzk)        │                       │           │
│                   안전필터 통과              감정태그 추출     │
│                         │                       │           │
│                      TTS 생성(ElevenLabs)        │           │
│                         │                       │           │
│                  [오디오 + 표정 이벤트] ─── WebSocket ──┐     │
└────────────────────────────────────────────────────┼────────┘
                                                      ▼
┌─────────────────────────────────────────────────────────────┐
│              Avatar Frontend (브라우저, TypeScript)          │
│  three-vrm 으로 VRM 렌더 · 오디오 재생 · 립싱크 · 표정 적용  │
└─────────────────────────────────────────────────────────────┘
                                                      │
                                          OBS 브라우저 소스로 캡처
                                                      ▼
                                          YouTube Live / 치지직 송출
```

### 구성요소별 기술 선택 (초안)
| 구성요소 | 기술 | 비고 |
|---|---|---|
| 두뇌(LLM) | 기존 OpenAI 연동 유지 | 페르소나/모델은 추후 비교·결정 |
| 음성(TTS) | ElevenLabs (`eleven_multilingual_v2`) | 기존 호출 코드 재활용. 한/영 모두 지원 |
| 아바타 렌더 | `@pixiv/three-vrm` (브라우저/TS) | OBS 브라우저 소스로 캡처 |
| 아바타 모델 | VRoid Studio 자체 제작 VRM | 무료·상업 이용 가능·내가 소유 |
| 채팅 수집 | YouTube Data/Live API → (이후) Chzzk | `ChatSource` 인터페이스로 추상화 |
| 백엔드↔프론트 | WebSocket | 오디오/표정 이벤트 전달 |
| 송출 | OBS Studio | 두 플랫폼 공통 |

> **왜 백엔드=Python, 프론트=TS 인가:** 아바타는 브라우저에서만 도므로 프론트는 TS가 불가피.
> 백엔드는 AI/채팅 라이브러리 생태계와 기존 Python 자산(TTS·LLM 코드) 때문에 Python으로 둔다.
> 둘은 WebSocket으로만 연결되어 서로 독립적으로 개발·교체 가능.

## 7. 로드맵 (보이는 결과물 단위로 분할)

| 단계 | 목표 | 완료 기준 (눈에 보이는 결과) |
|---|---|---|
| **0. 아바타 띄우기** | VRM을 브라우저에 렌더 | 화면에 AiCy가 뜨고, 오디오에 입이 맞춰 움직임 |
| **1. 오프라인 MVP** | 입력→LLM→TTS→말하기 | 키보드로 질문하면 아바타가 자기 목소리로 답하며 움직임 (방송 없음) |
| **2. 라이브 채팅 연동** | YouTube 채팅 자동 반응 | 유튜브 라이브 채팅을 읽고 자동으로 대답 |
| **3. 실제 송출** | OBS로 라이브 | 실제 YouTube Live에 AiCy가 방송 중 |
| **4. 운영 고도화** | 기억·idle·안전필터·도네 | 장시간 무인 방송이 안정적으로 유지됨 |
| **5. 한국어/치지직 확장** | 플랫폼·언어 전환 | 설정만 바꿔 치지직 한국어 방송 가동 |

## 8. 기존 코드 처리 방침

- **정체성·노하우 유지:** AiCy 컨셉, ElevenLabs TTS 호출, 대화기록 유지, OpenAI 호출 패턴.
- **구조는 재설계:** Discord 전용 구조·하드코딩 경로·죽은 주석·중복 페르소나 정리.
- **방법:** 기존 파일을 삭제하지 않고 `legacy/` 폴더로 이동해 참고용 보관 → 새 구조에서 필요한 부분만 이식.

## 부록 A. 자주 쓸 용어
- **VRM:** 3D 아바타용 오픈 파일 표준. VRoid로 생성, three-vrm으로 제어.
- **립싱크(viseme):** 음성의 발음에 맞춰 입모양 파라미터를 바꾸는 것.
- **idle 행동:** 채팅이 없을 때 캐릭터가 혼잣말·움직임으로 빈 시간을 채우는 것.
- **OBS 브라우저 소스:** 웹페이지를 방송 화면에 합성하는 OBS 기능. 아바타를 송출에 얹는 통로.
