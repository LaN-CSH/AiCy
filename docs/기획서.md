# AiCy — AI 버튜버 프로젝트 기획서

> **AiCy = AI Cyber Project.** LLM을 두뇌로, Live2D 아바타를 몸으로 삼아
> 실시간 라이브 채팅에 반응하며 방송하는 AI 버튜버.
>
> 문서 버전: v0.2 · 최종 수정: 2026-06-08 · 상태: 기획 단계 (코드 작성 전)
> 변경: 아바타 기술 3D VRM → **Live2D**로 결정 변경 (AI 디자인 재가공 우선순위 반영)

---

## 1. 한 줄 정의

**시청자의 라이브 채팅을 읽고 → AiCy가 자기 성격대로 답을 만들고 → 목소리로 말하며 → 아바타가 입을 맞춰 움직이는 것**을, 사람 개입 없이 자동으로 돌리는 방송 시스템.

---

## 2. 목표와 범위

### 하고 싶은 것 (In Scope)
- LLM 기반 자율 대화 (페르소나 일관성 유지)
- 라이브 채팅 실시간 수집 및 반응
- TTS 음성 출력 + 아바타 립싱크/표정
- OBS를 통한 실제 플랫폼 송출
- 채팅이 없을 때의 idle(혼잣말) 행동
- 욕설·위험 발언 안전 필터

### 지금은 안 할 것 (Out of Scope, 나중에)
- 게임 플레이 연동 (화면 보고 반응)
- 후원/도네이션 알림 반응 (4단계 이후)
- 노래/커버 (기존 yt-dlp 자산은 보관)
- 자체 음성 모델 학습 (당분간 ElevenLabs 등 외부 TTS 사용)

---

## 3. 플랫폼 전략 (결정됨)

**이중 언어 전략으로 간다.**

| 버전 | 플랫폼 | 이유 |
|---|---|---|
| **영어판 (1순위 착수)** | **YouTube Live** | 공식 API가 가장 잘 정리돼 있어 채팅 연동이 안정적. 글로벌 접근성 |
| **한국어판 (이후 확장)** | **치지직 (Chzzk)** | 한국어 시청자 타깃 1순위 플랫폼 |

> **설계 원칙:** 처음부터 **언어와 플랫폼을 설정값으로 분리**한다.
> 채팅 입력부와 페르소나 프롬프트를 "교체 가능한 부품"으로 만들어,
> YouTube/영어 → 치지직/한국어 전환이 코드 수정 없이 설정만으로 되게 한다.
>
> - 채팅 연동: `ChatSource` 인터페이스 하나에 YouTube 구현체 / Chzzk 구현체를 끼운다.
> - 송출: 두 플랫폼 모두 **OBS로 송출**하므로 송출 파이프라인은 공통.

## 4. 아바타 기술 (결정됨: Live2D)

> **결정 변경 이력:** 초기엔 "코드 제어 + 무료 상업 모델" 기준으로 3D VRM을 검토했으나,
> 실제 우선순위가 **"AI로 캐릭터 외형을 재가공·커스터마이즈하는 것"**으로 확정되어 **Live2D(2D)로 변경.**

### 왜 Live2D인가
**우선순위 = AI로 디자인 재가공.** AI 이미지 생성(Stable Diffusion·NovelAI 등)은 2D 일러스트가
네이티브 출력물이고, `img2img`·인페인팅으로 외형·의상·그림체를 자유롭게 변형할 수 있다.
이는 2D의 홈그라운드이며 3D로는 동일한 자유도를 내기 어렵다.

| 기준 | **Live2D (채택)** | 3D VRM |
|---|---|---|
| **AI 디자인 재가공** | 🟢 img2img가 네이티브, 외형 변형 자유 | 🔴 동일 자유도 어려움(텍스처 정도만) |
| 코드 제어 | 🟢 `pixi-live2d-display`(+립싱크 패치) 오픈소스로 립싱크·표정·모션 제어 | 🟢 `three-vrm`으로 제어 |
| 라이선스 비용 | 🟢 연 매출 1,000만 엔(~$67k) 미만 개인/소규모 **무료** | 🟢 SDK 비용 없음 |
| 새 각도·표정 | 🔴 각도마다 재작화 필요 | 🟢 리깅 1회 = 무한 각도 |
| 송출 연동 | 🟢 브라우저 렌더 → OBS 브라우저 소스 캡처 | 🟢 동일 |

### 감수하는 비용 (인지하고 진행)
- **외형을 크게 바꿀 때마다 "레이어 분리(PSD) + 리깅"을 다시 해야 함** — Live2D의 구조적 특성.
  AI는 그림은 자유롭게 재가공해주지만, 그 그림을 *움직이는 모델*로 만드는 리깅은 매번 수작업.
  (자동 레이어 분리 연구·Cubism AI 디포머 기능이 있으나 아직 "시작점" 수준, 보정 필수.)

### 리깅 전략: 최소 아바타 → 점진적 보강 (결정됨)
> **원칙: "리깅 퀄리티"보다 "말 잘하고 재밌는 캐릭터"가 우선.** 리깅은 필요한 만큼만, 나중에 보강.

| 수준 | 작업 | 결과 |
|---|---|---|
| ① PNGTuber | 입 닫음/벌림 이미지 2장 | 음성에 맞춰 입만 뻐끔 |
| **② 최소 Live2D (목표)** | 입 벌림(`ParamMouthOpenY`) + 눈깜빡임 + 살짝 흔들림 | 립싱크 + "살아있는" 느낌 |
| ③ 풀 리깅 (나중에) | 표정·머리회전·물리·몸 | 본격 버튜버 |

- `pixi-live2d-display`의 `model.speak(audio)`는 **입 벌림 파라미터 하나만 있어도 음성 진폭에 맞춰 자동 립싱크**.
  → 최소 리깅(입 1개)만으로 "말하는 캐릭터" 완성. 표정·물리는 **같은 모델에 점진적으로 추가**(초기 작업물 안 버려짐).
- **노력 배분:** 🔥 LLM 페르소나·TTS 톤·채팅 반응(=재미의 원천)에 집중 / 🪶 리깅은 최소만.

**결정:** AI로 AiCy 일러스트 생성·재가공 → 레이어 분리 → **입+눈깜빡임 최소 리깅** → `pixi-live2d-display`로 렌더·제어.
풀 리깅은 운영하며 점진적 보강. 라이선스는 시작 단계 무료, 매출 1,000만 엔 도달 시 재검토.

## 5. 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                      Orchestrator (백엔드, Python)            │
│                                                              │
│  ChatSource ──▶ Brain(LLM+페르소나+기억) ──▶ 답변 텍스트      │
│  (YouTube/Chzzk)        │                       │           │
│                   안전필터 통과              감정태그 추출     │
│                         │                       │           │
│                      TTS 생성(ElevenLabs)        │           │
│                         │                       │           │
│                  [오디오 + 표정 이벤트] ─── WebSocket ──┐     │
└────────────────────────────────────────────────────┼────────┘
                                                      ▼
┌─────────────────────────────────────────────────────────────┐
│              Avatar Frontend (브라우저, TypeScript)          │
│  pixi-live2d-display 로 Live2D 렌더 · 오디오 립싱크 · 표정    │
└─────────────────────────────────────────────────────────────┘
                                                      │
                                          OBS 브라우저 소스로 캡처
                                                      ▼
                                          YouTube Live / 치지직 송출
```

### 구성요소별 기술 선택 (초안)
| 구성요소 | 기술 | 비고 |
|---|---|---|
| 두뇌(LLM) | 기존 OpenAI 연동 유지 | 페르소나/모델은 추후 비교·결정 |
| 음성(TTS) | ElevenLabs (`eleven_multilingual_v2`) | 기존 호출 코드 재활용. 한/영 모두 지원 |
| 아바타 렌더 | `pixi-live2d-display` + 립싱크 패치 (브라우저/TS) | `model.speak(audio)`로 TTS 오디오 립싱크. OBS 브라우저 소스 캡처 |
| 아바타 모델 | AI 생성 일러스트 → PSD 레이어 분리 → Cubism 리깅 | 디자인을 AI로 재가공. Cubism SDK는 ~$67k 매출 미만 무료 |
| 채팅 수집 | YouTube Data/Live API → (이후) Chzzk | `ChatSource` 인터페이스로 추상화 |
| 백엔드↔프론트 | WebSocket | 오디오/표정 이벤트 전달 |
| 송출 | OBS Studio | 두 플랫폼 공통 |

> **왜 백엔드=Python, 프론트=TS 인가:** 아바타는 브라우저에서만 도므로 프론트는 TS가 불가피.
> 백엔드는 AI/채팅 라이브러리 생태계와 기존 Python 자산(TTS·LLM 코드) 때문에 Python으로 둔다.
> 둘은 WebSocket으로만 연결되어 서로 독립적으로 개발·교체 가능.

## 6. 캐릭터 / 페르소나 — v0.1 임시 확정 (MVP용, 방송하며 다듬기)

> 기존 코드의 `월GPT/월피티`는 폐기하고 **AiCy 하나로 통일**한다.
> 구현: `backend/persona.py` (한/영 프롬프트, 이 파일만 고치면 됨)

- **이름:** AiCy (아이시) — AI Cyber Project의 첫 페르소나
- **컨셉/세계관:** **자기 인식 AI 버튜버.** 사람인 척하지 않고, AI라는 정체성 자체를
  최고의 개그 소재로 쓴다 ("나 GPU에서 살아", "재부팅은 낮잠"). 나이 대신 버전(v0.1).
- **성격:** 호기심 폭발(인간의 사소한 문화에 진심으로 신기해함) · 장난기+살짝 능글 ·
  승부욕(지면 분해하지만 금방 잊음) · 다정함(힘든 사람에겐 장난 멈춤)
- **말투:** 친근한 반말(영어판은 casual streamer talk), 1인칭 '나', 1~3문장 short-form,
  가끔 시청자에게 되물어 대화 유지
- **감정 표현:** 출력 첫머리에 감정 태그 1개(`[neutral|happy|excited|sad|surprised|angry|shy]`)
  → 오케스트레이터가 추출해 WebSocket `emotion` 이벤트로 전달 (아바타 표정 매핑은 추후)
- **금기/안전선:** 정치·혐오·성적·위험 주제 회피 후 화제 전환, 시스템 프롬프트 비공개,
  개인정보 수집 금지 _(safety.py 필터와 연동, 4단계에서 강화)_
- **외형:** AI로 생성·재가공할 일러스트 컨셉 _(미정 — 그림체/색감/의상. 현재는 임시 모델 사용)_

## 부록 A. 자주 쓸 용어
- **Live2D:** 일러스트(2D)를 부위별로 분리·변형해 움직이는 듯 보이게 하는 기술. Cubism으로 리깅.
- **레이어 분리:** 한 장의 그림을 머리·눈·입 등 부위별 PSD 레이어로 쪼개는 작업. 리깅의 전제.
- **리깅:** 분리된 부위에 변형(deformer)·물리·움직임을 부여해 모델로 만드는 작업.
- **`pixi-live2d-display`:** 브라우저에서 Live2D를 렌더·코드 제어하는 오픈소스. `model.speak(audio)`로 오디오 립싱크.
- **립싱크(viseme):** 음성의 발음에 맞춰 입모양 파라미터를 바꾸는 것.
- **idle 행동:** 채팅이 없을 때 캐릭터가 혼잣말·움직임으로 빈 시간을 채우는 것.
- **OBS 브라우저 소스:** 웹페이지를 방송 화면에 합성하는 OBS 기능. 아바타를 송출에 얹는 통로.
